8. zabezpieczenie

  1. Do pliku movie.component.ts dołożyć 
    
                            import { DomSanitizer } from '@angular/platform-browser';
  2. Zmodyfikować dalszą część: 
    
                            private sanitizer: DomSanitizer;

                            constructor(sanitizer: DomSanitizer) {
                                this.sanitizer = sanitizer;
                            }
  3. Dodać też funkcję getWrappedMovieUrl: 
    
                        getWrappedMovieUrl() {
                            return this.sanitizer.bypassSecurityTrustResourceUrl(this.movie.url);
                        }
  4. W movie.component.html zmienić src: 
    
                        [src]="getWrappedMovieUrl()"
UWAGA! HARDKOR!
Pole prywatne = używane tylko w klasie
Wstrzykiwanie zależności - serwis jest dodany przez Angulara
[src] bo nie-string; teraz mamy obiekt, który mówi, że url bezpieczny

Bonusy

  1. Zamiast pisać: 
    
                            private sanitizer: DomSanitizer;

                            constructor(sanitizer: DomSanitizer) {
                                this.sanitizer = sanitizer;
                            }
    można: 
    
                            constructor(private sanitizer: DomSanitizer) {}
  2. Podobnie get jest częstym tematem i zamiast: 
    
                        getWrappedMovieUrl() { /* ... */ }
    można: 
    
                        get wrappedMovieUrl() { /* ... */ }
    i później: 
    
                        [src]="wrappedMovieUrl"
service, component, module
