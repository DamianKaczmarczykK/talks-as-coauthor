Dlaczego React?

  • Przełomowy, rozwijany. Tworzony reaktywnie :)
  • React 16 = gruntowne zmiany, a API bez zmian
  • Async Rendering = Time Slicing + Suspense
  • W Preact'cie wymieniony Suspense
  • W Facebooku lubią przełomowe rzeczy. Aż nawet JSON-a porzucili w GraphQL-u; może aż za hardkorowo
  • Dan Abramov: We have built a generic way for components to suspend rendering while they load asynchronous data
  • Twitter jest fajny, bo trzeba bardzo zwięźle zbierać myśli
  • Pauzujemy (eng. suspend) renderowanie - hej, trzeba doczytać trochę danych
  • Było przerywanie potrzebne - interrupting - i do time slicingu i do suspend
  • Suspend na razie tylko z lazy loadingiem komponentów działa (https://reactjs.org/docs/react-api.html#suspense)
  • https://medium.com/@baphemot/understanding-react-suspense-1c73b4b0b1e6
  • Jeszcze dużo się tu pozmienia
