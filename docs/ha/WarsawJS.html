TL;DR

  • Actions API 
    
                            const IncBy = (state, { number }) => ({ ...state, val: state.val + number });
                            <button onClick={[IncBy, { number: 5 }]}>Add 5</button>
  • Effects API
    • Async (HTTP, routing, opóźnianie)
  • Subscriptions API
    • Globalne zdarzenia (np. od window)
  • Lazy Lists/components
    • Render tylko na zmianę konkretnej rzeczy w stanie
    • <Lazy render={ListView} list={state.list} />
  • Middleware/DevTools
  • Akcje w v2 są deklaratywne, nie mówimy kiedy, tylko gdzie (wcześniej i kiedy i gdzie)
  • Czyste akcje, sami mergujemy stan
  • Akcja zwraca nowy, globalny stan i efekt
  • Brak "globalnych" akcji - są gdzie trzeba
  • Efekty = komendy z Elma (commands)
  • Efekt = obiekt reprezentujący side effect
  • Lazy ~ takie trochę memo z Reacta
  • Middleware - nie tak jak Redux, tylko dev toolsy
  • Tryb dev = wincyj logów, time-travel, warningi
